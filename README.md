## 🧠 Как это работает

1. **Захват экрана** — библиотека `dxcam` вырезает центральную область (FOV).
2. **Детекция целей** — YOLOv11 (TensorRT) определяет людей, вычисляет координаты головы.
3. **Адаптивная рандомизация** — точка прицеливания смещается в пределах динамического хитбокса (зависит от размера цели).
4. **Имитация человеческого поведения**:
   - Случайная задержка реакции (50–250 мс)
   - Вероятность промаха
   - Плавное сглаживание с джиттером
5. **Аппаратное движение мыши** — координаты отправляются по Serial на Arduino, которая эмулирует USB-мышь.

## 🖥️ Необходимое оборудование

| Компонент | Рекомендации |
|-----------|--------------|
| **Arduino** | Leonardo / Pro Micro (ATmega32U4) — наличие встроенного USB-HID |
| **Видеокарта** | NVIDIA RTX 20/30/40/50 (для TensorRT) |
| **ПК** | Intel i5+/Ryzen 5+, 16+ GB RAM |

> ⚠️ **Важно:** Без NVIDIA GPU с поддержкой TensorRT производительность YOLO будет недостаточной.

## 📦 Готовый исполняемый файл (Release)

Если вы не хотите возиться с Python и зависимостями, скачайте последнюю версию из [Releases](https://github.com/ваш-репозиторий/releases).  
Архив содержит:

- `StealthAimer.exe` — основное приложение.
- Папка `resources/` с прошивкой `firmware.hex` и утилитой `avrdude`.

**Порядок использования:**
1. Подключите Arduino к ПК.
2. Запустите `StealthAimer.exe`.
3. Вкладка **Прошивка** → выберите COM-порт → **Прошить** (только при первом использовании).
4. Получите модель `.engine` (см. раздел [Подготовка модели](#-подготовка-модели-yolo-engine)).
5. Вкладка **Настройки** → укажите путь к `.engine` и COM-порт.
6. Вкладка **Управление** → **Запустить аим** (или нажмите F1).

🎯 Подготовка модели YOLO (.engine)
Модель в формате .engine необходима для работы на GPU. Если у вас есть NVIDIA RTX, выполните:

from ultralytics import YOLO
model = YOLO("yolo11m.pt")
model.export(format="engine", half=True, imgsz=640)

Саму модель загрузить можно отсюда yolo predict model=yolo11m.pt source='https://ultralytics.com/images/bus.jpg'


🔌Прошивка Arduino
Первый способ (через GUI):

Подключите Arduino, выберите COM-порт во вкладке Прошивка, нажмите Прошить.

Второй способ (вручную):

Откройте файл arduino_mouse.ino в Arduino IDE.

Выберите плату: Инструменты → Плата → Arduino Leonardo (или Pro Micro).

Нажмите Загрузка.

После прошивки светодиод на плате мигнёт три раза — устройство готово.





🚀 Запуск и настройка
Запустите StealthAimer.exe.

Вкладка Прошивка — если ещё не прошивали, выберите порт и прошейте.

Вкладка Настройки:

Модель: выберите yolo11m.engine.

COM-порт: выберите порт Arduino (обычно тот же, что использовали для прошивки).

Настройте параметры аима (FOV, сглаживание, вероятность промаха и т.д.).

Вкладка Управление → нажмите Запустить аим (или F1). Для остановки — Остановить (или F2).

Программа начнёт захват экрана, и при появлении цели мышь будет двигаться автоматически.



🎛️ Настройка параметров
Параметр	Описание	Рекомендации
Confidence	Уверенность детекции (0–1)	0.45–0.55
FOV (px)	Область сканирования (квадрат)	300–500 (в зависимости от разрешения)
Smooth	Сглаживание движения (0–1)	0.2–0.4 для быстрых игр, 0.5+ для плавного
Hitbox size	Разброс точки попадания (% от ширины цели)	1–3%
Miss chance	Вероятность промаха (0–1)	0.05–0.1





🔧 Устранение неполадок
Arduino не двигает мышь

Убедитесь, что выбран правильный COM-порт в настройках.

Проверьте, что плата прошита (мигает светодиод).

Попробуйте перезагрузить программу или переподключить Arduino.

Низкий FPS детекции

Убедитесь, что используется модель .engine, а не .pt.

Уменьшите imgsz в коде (например, 512).

Проверьте, что видеокарта поддерживает TensorRT.

Ошибка при запуске аима

Проверьте, что файл .engine существует и доступен.

Убедитесь, что выбран правильный COM-порт.
